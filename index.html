Автоматизация

ГОСТ 34, РД 50, ЕСКД

Проектная документация по стандарту

Разработаем необходимую проектную документацию по Российским государственным стандартам, учитывая требования ГОСТ 34, РД 50 и ЕСКД.

Создадим необходимый Вам набор проектных документов, чертежей, принципиальных схем. Разработаем эксплуатационную документацию. Поможем в написании технического задания.

ОВЕН, WAGO, Segnetics

Сертифицированное оборудование

Для построения систем АСУ ТП мы используем cертифицированное оборудование известных производителей. От доступных решений на ОВЕН, до надежных и приемлемых по стоимости WAGO и Segnetics.

Организуем поставку любого оборудования для построения АСУ ТП. Контроллеры, регуляторы, устройства ввода-вывода, измерительные преобразователи, устройства связи и другое оборудование.

Программируем контроллеры в среде CoDeSys. Настраиваем сетевые интерфейсы для взаимодействия существущего оборудования с контроллерами.

Щиты автоматики, кабельные трассы

Монтаж, пусконаладочные работы и сопровождение

Собираем щиты автоматики соблюдая правила ПУЭ, СНиП, ПТЭЭП. Производим монтаж кабельных трасс. Подключаем оборудование.

Производим пусконаладочные работы. Осуществляем дальнейшее сопровождение.

ПРОСТОЕ ЛИЦЕНЗИРОВАНИЕ

Лицензирование без ограничений

Аппаратные ключи не требуются. Используйте систему без ограничений в рамках предприятия.

ЭКОНОМИЯ НА ОБОРУДОВАНИИ

Используйте существующее оборудование

Для организации рабочего места нужен только Web-браузер. Данный факт позволяет запускать Web SCADA на существующем оборудовании.

ПРОСТОЕ СОПРОВОЖДЕНИЕ

Снижение издержек на сопровождение

Быстрое добавление нового рабочего места. После добавления достаточно запустить Web-браузер и осуществить вход.

Обновление системы до новой версии происходит сразу на всех рабочих местах и занимает минимум времени.

ОБЛАЧНАЯ СЛУЖБА

Размещайте Web SCADA в облаке

Размещение Web SCADA в облаке позволяет снизить издержки на приобретение серверного оборудования.

Безопасно

Web SCADA использует проверенные решения по безопасности Web-систем.

ГИБКОЕ УПРАВЛЕНИЕ ДОСТУПОМ

Разграничение доступа

Доступ к системе предоставляется зарегистрированным пользователям.

Защита пользователей осуществляется при помощи паролей или сертификатов.

Настройка доступа к функциям системы производится на основе ролей. Например, роли для оператора, инженера, менеджера, администратора.
